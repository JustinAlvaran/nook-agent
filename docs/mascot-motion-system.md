# Mascot motion system

## Implemented

Task data flows through a pure brain-state mapper and motion signal before reaching `Nook3D`. The signal contains state, intensity, focus target, expression, and progress. Running no longer means walking. Manual dragging is disabled by default, and reduced-motion preferences disable procedural cue animation.

## Partially implemented

The current procedural model supports breathing, head motion, arm gestures, error and sleep poses. Predefined stage anchors, transition validation, visibility pausing, and complete visual snapshots remain Phase E work.

## Future desktop support

Desktop presentation commands will remain separate from computer-control permissions. No unrestricted operating-system movement or control is implemented.
