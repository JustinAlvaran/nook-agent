# MVP scope and product truth

The MVP proves one thing well: a user can ask their Nook for useful work, inspect the exact bounded plan/tool, approve a reversible Nook-only change, execute once, see verified output, refresh, and recover the same timeline.

Supported examples:

- “Draft launch copy and a checklist.”
- “Guide me through creating a Facebook Page.” Nook asks for missing business details and stops before Facebook.
- “Open Facebook.” Nook prepares a fixed user-clicked link; it does not navigate autonomously.
- “Make my Nook more concise.” Nook shows a permission ticket, then changes one setting after approval.

Explicitly excluded: personal Facebook registration; autonomous browsing or form submission; screen reading; arbitrary desktop control; email/post/file/calendar writes; purchases, publishing, deletion, permissions, or account changes; creator code uploads; paid checkout; claims that a procedural mascot is a production rigged asset.

Marketplace, connectors, and desktop sections remain preview/foundation surfaces. They must not imply installed permissions, earnings, ratings, live activity, or successful external actions.
