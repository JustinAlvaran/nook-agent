# Marketplace data boundaries

Marketplace catalog, order, entitlement, loadout, and creator records never authorize agent tools. Cosmetics are visual only. A future behavior pack may configure declared orchestration values but cannot contain arbitrary executable code or credentials. Capability grants live separately and require their own consent and policy.

The current platform-owned preview catalog can grant only verified free entitlements. Paid ownership must come only from idempotently verified Stripe webhooks, never redirect parameters. Creator selling remains disabled until asset validation, license evidence, moderation, immutable versions, seller verification, payouts, recalls, refunds/disputes, and support policy exist.
